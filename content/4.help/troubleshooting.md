## 安装问题

### 安装进度卡在 45%？

这是正常现象。安装过程中需要创建 Python 虚拟环境并从 PyPI 安装 QwenPaw 及其依赖，此阶段耗时较长，请耐心等待。具体耗时取决于网络状况和 NAS 性能，通常 5-15 分钟。

### 安装失败怎么办？

1. 检查 fnOS 版本是否 >= 0.9.21
2. 确保 NAS 可以正常访问外网（PyPI 源）
3. 确保 NAS 有足够的磁盘空间（建议至少 5GB 可用空间）
4. 尝试卸载后重新安装

## 启动问题

### 点击图标无法打开控制台？

1. 确认 QwenPaw 服务是否已启动（查看飞牛应用状态）
2. 检查端口 19091 是否被其他应用占用
3. 尝试重启 QwenPaw 应用
4. 如果仍无法打开，在飞牛文件管理器中检查日志文件

### 启动后页面空白或加载失败？

1. 等待几秒后刷新页面
2. 清除浏览器缓存
3. 尝试使用无痕模式访问
4. 检查浏览器控制台是否有报错信息

## 连接问题

### Ollama 连接失败？

1. 确认 Ollama 应用已在飞牛应用商店安装并启动
2. 检查 Ollama 服务地址是否正确
3. API Key 字段不能为空，填写任意非空字符串（如 `ollama`）
4. 确认 Ollama 中已下载模型（在 Ollama 应用中操作）

### 模型 API 连接超时？

1. 检查 API 地址是否可以从 NAS 访问
2. 检查 API Key 是否正确
3. 确认 NAS 可以访问外网（如果使用云端 API）
4. 尝试在 NAS 上 `curl` 测试 API 地址

## 通道问题

### QQ 通道配置相关？

QQ 通道基于 OneBot 协议。你需要先搭建 OneBot 服务（如 napcat、LLOneBot 等），然后在 QwenPaw 中配置 WebSocket 连接地址。

### 微信通道无法登录？

1. 确认微信账号状态正常
2. 尝试重新扫码登录
3. 部分情况下需要在新设备上验证，请按微信提示操作

> **注意**：使用个人微信接入自动化服务存在账号风险，请自行评估。

### 通道连接断开怎么办？

- 大部分通道支持自动重连，等待片刻即可恢复
- 如长时间未恢复，尝试在通道页面手动重新连接
- OneBot WebSocket 断连会自动重启恢复

## 性能问题

### 控制台响应慢？

1. 检查 NAS 的 CPU 和内存使用情况
2. 如果使用了本地大模型，模型推理需要消耗大量计算资源
3. 建议为 QwenPaw 所在 NAS 分配足够的硬件资源

### 内存占用过高？

1. 减少同时运行的智能体数量
2. 关闭不使用的通道连接
3. 清理不需要的文件和备份
